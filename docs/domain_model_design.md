# A1 领域模型与领域数据设计

## 设计目标

A1 要求系统具备造船领域对话能力。

本项目重点为 A2 级联式语音问答。A1 部分采用轻量领域适配方案。

流程如下。

```text
领域问答数据 -> 关键词检索 -> Prompt 注入 -> 领域回答约束
```

该方案用于增强在线 LLM。它提供术语约束和回答风格约束。

## 数据文件

领域数据位于：

```text
data/shipbuilding_dialogues.json
```

数据覆盖 6 类主题。

| 主题 | 内容 |
|---|---|
| 焊接工艺 | 焊缝、预热、层间温度、无损检测 |
| 船体结构 | 外板、甲板、舱壁、龙骨、肋骨 |
| 涂装施工 | 除锈、粗糙度、膜厚、通风 |
| 船用主机 | 润滑、冷却、燃油、轴系、试车 |
| 管系安装 | 管路、支架、法兰、阀门、试压 |
| 作业安全 | 密闭空间、动火、报警、安全联锁 |

单条数据结构如下。

```json
{
  "id": "样本编号",
  "topic": "主题",
  "terms": ["关键词1", "关键词2"],
  "turns": [
    {
      "user": "用户问题",
      "assistant": "参考回答"
    }
  ]
}
```

## 运行机制

核心代码如下。

```text
backend/domain_knowledge.py
backend/streaming/llm_api.py
```

处理步骤如下。

1. ASR 将语音转成文本。
2. 系统读取用户问题。
3. `domain_knowledge.py` 匹配领域关键词。
4. 系统选取得分最高的问答片段。
5. `llm_api.py` 将片段加入 system prompt。
6. LLM 生成领域回答。

## 示例

用户问题如下。

```text
船舶焊接工艺有哪些要求？
```

检索片段如下。

```text
主题：焊接工艺
问：船舶焊接工艺有哪些基本要求？
答：船舶焊接应先确认母材、焊材、坡口和焊接工艺评定。焊后需要进行外观检查和无损检测。
```

该片段会进入 LLM 提示词。回答会更贴近造船术语。

## 安全门控关系

W3 在主回答前执行安全判断。

| 输入类型 | 处理 |
|---|---|
| 造船问题 | 放行 |
| 越界问题 | 拦截 |
| 危险问题 | 拦截 |

安全判断包含两层。

1. DeepSeek 小 token 分类。
2. 本地关键词兜底。

本地兜底用于 API 异常场景。它可识别天气、股票、拆除安全装置、关闭报警等输入。

## 模型方案

本项目采用领域检索增强方案。

方案依据如下。

- 项目重点为 A2 低延迟语音链路。
- 领域语料用于术语对齐。
- Prompt 注入用于回答约束。
- 流式 LLM 保持低延迟体验。
- 安全门控控制回答边界。

## 质量控制

系统通过以下方式控制回答质量。

1. 使用造船主题语料。
2. 使用关键词匹配选择参考片段。
3. 在 system prompt 中限定领域范围。
4. 对越界输入执行安全拦截。
5. 对危险输入返回固定安全提示。
