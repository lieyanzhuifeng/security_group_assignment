# W1 vs W2 首段音频延迟对比 — 实验报告

## 实验设计

| 项目 | 说明 |
|------|------|
| 度量指标 | **首段音频延迟**（first_audio）：音频就绪 → 首段 TTS 音频合成完成 |
| 测试用例 | 4 组船舶建造领域 WAV 录音（123~157 KB） |
| 重复次数 | 每用例 × 3 次，取平均 |
| W1 管线 | ASR → 完整 LLM → 完整 TTS（顺序执行） |
| W2 管线 | ASR → 流式 LLM 逐 token → 逐句切分 → 逐句 TTS（并行） |

## 实验结果

| 用例 | 描述 | W1 (s) | W2 (s) | 加速倍数 | 节省时间 |
|------|------|--------|--------|----------|----------|
| t01 | 焊接工艺要求 | 58.88 | 15.98 | **3.7×** | 42.9s |
| t02 | 船体结构 | 30.15 | 19.76 | **1.5×** | 10.4s |
| t03 | 涂装要求 | 47.25 | 20.07 | **2.4×** | 27.2s |
| t04 | 主机型号 | 41.12 | 18.21 | **2.3×** | 22.9s |
| **平均** | | **44.35** | **18.50** | **2.4×** | **25.8s** |

## 关键观测

**1. W2 首段音频延迟稳定在 15~20s 区间**

W2 的首句触发时间受 ASR 时长（2~5s）+ 首句 LLM token 到达 + 首句 TTS 合成决定，波动较小。4 个用例的 W2 `first_audio` 在 10.8s ~ 25.2s 之间。

**2. W1 延迟随 LLM 回答长度剧烈波动**

W1 必须等完整 LLM 生成 + 完整 TTS 合成，延迟取决于回答长度。t01 的 W1 达到 58.88s（LLM 返回了最长回答），而 t02 仅 30.15s。W2 不受全量回答长度影响。

**3. t01 加速最显著（3.7×）**

焊接工艺要求触发了 LLM 最长响应，W1 需等待全部 token 生成完毕才启动 TTS，而 W2 在首句完成后即开始播放，节省了约 43 秒。

**4. t02 加速幅度最小（1.5×）**

船体结构问题回答较短，LLM 全量生成时间本身不长，W1 等待的额外时间有限，流式优势相对小。

## 结论

W2 全流式管线在首段音频响应上相比 W1 基线平均加速 **2.4 倍**，最差情况也有 1.5 倍改善。核心收益来自将 TTS 启动时机从"全量 LLM 完成"提前到"首句 LLM 完成"，消除了等待完整回答的无效延迟。
